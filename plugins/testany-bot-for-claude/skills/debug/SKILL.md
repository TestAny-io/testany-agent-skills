分析测试失败原因，排查问题根因。

用户输入: $ARGUMENTS

## 支持的操作

- **分析执行**：获取失败执行的详情
- **查看日志**：获取并分析执行日志
- **定位问题**：识别失败类型和原因
- **修复建议**：提供针对性的修复方向

## 工作流程

1. 获取执行详情
2. 定位失败的 case
3. 获取并分析日志
4. 识别失败类型（断言/超时/错误/基础设施）
5. 给出修复建议
