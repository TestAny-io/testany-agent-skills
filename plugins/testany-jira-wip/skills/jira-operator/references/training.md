# 培训设计参考

## 前置条件

- 用户已确认培训对象（新员工/现有员工/团队）
- 用户已确认培训时长
- 用户已确认培训内容范围

## 培训类型

### 新员工入职培训

**目标**：让新员工快速上手 Jira/Confluence

**培训时长**：1-2 小时

**培训内容**：

| 章节 | 内容 | 时长 |
|------|------|------|
| Jira 概览 | Jira 是什么、为什么使用 | 10 分钟 |
| 账号和登录 | 创建账号、登录、密码重置 | 5 分钟 |
| 创建 Issue | 如何创建 Issue、填写字段 | 15 分钟 |
| 查询 Issue | JQL 基础、Filter 使用 | 20 分钟 |
| 更新 Issue | 添加评论、修改字段、状态变更 | 15 分钟 |
| 看板使用 | 如何使用看板、拖拽操作 | 15 分钟 |
| Confluence 使用 | 查看文档、编辑页面 | 10 分钟 |
| Q&A | 答疑 | 20 分钟 |

### 进阶培训

**目标**：提升团队 Jira/Confluence 使用效率

**培训时长**：2-3 小时

**培训内容**：

| 章节 | 内容 | 时长 |
|------|------|------|
| JQL 高级查询 | 复杂条件、时间函数、子查询 | 30 分钟 |
| 自动化规则 | Jira Automation 基础 | 30 分钟 |
| 仪表板配置 | 创建仪表板、添加 Gadget | 30 分钟 |
| 工作流理解 | 理解工作流、状态流转 | 20 分钟 |
| Confluence 高级功能 | 页面模板、标签体系 | 20 分钟 |
| 最佳实践 | 常见错误、效率技巧 | 30 分钟 |

### 管理员培训

**目标**：让管理员能够维护和管理 Jira/Confluence

**培训时长**：4 小时

**培训内容**：

| 章节 | 内容 | 时长 |
|------|------|------|
| 工作流设计 | 创建工作流、配置状态流转 | 60 分钟 |
| 权限管理 | 角色定义、权限方案配置 | 60 分钟 |
| 项目模板 | 创建项目模板、配置默认值 | 30 分钟 |
| 自动化规则 | 高级自动化、ScriptRunner | 60 分钟 |
| DevOps 集成 | CI/CD 集成、Webhook 配置 | 30 分钟 |

## 培训材料

### 入职指南模板

```markdown
# Jira 入职指南

## 什么是 Jira？
Jira 是我们团队的项目管理和问题跟踪工具。

## 如何登录？
1. 访问 https://company.atlassian.net
2. 使用公司邮箱登录
3. 首次登录需要设置密码

## 创建 Issue

### 创建 Story
1. 点击 "Create"
2. 选择 "Story"
3. 填写 Summary（一句话描述）
4. 填写 Description（详细描述）
5. 点击 "Create"

### 创建 Bug
1. 点击 "Create"
2. 选择 "Bug"
3. 填写 Summary、Description
4. 设置优先级（High/Medium/Low）
5. 点击 "Create"

## 查询 Issue

### 我的任务
- Filter: `assignee = currentUser() AND status in ("To Do", "In Progress")`

### 本 Sprint 的任务
- Filter: `project = PROJ AND sprint in openSprints()`

## 更新 Issue

### 添加评论
1. 打开 Issue
2. 在 "Comment" 输入框输入评论
3. 点击 "Comment"

### 更改状态
1. 打开 Issue
2. 点击 "Status" 下拉框
3. 选择新状态（如 "In Progress"）

## 常见问题

### Q: 如何找回密码？
A: 点击登录页面的 "Forgot password"

### Q: 如何分配 Issue？
A: 在 Issue 页面点击 "Assignee" 字段，选择人员

### Q: 如何查看 Sprint 信息？
A: 点击 "Sprint" 字段，查看 Sprint 详情
```

### 快速参考卡片

**常用快捷键**：

| 快捷键 | 功能 |
|--------|------|
| c | 创建 Issue |
| . | 快速搜索 |
| ? | 显示所有快捷键 |
| j | 下一个 Issue |
| k | 上一个 Issue |

**常用 JQL 查询**：

| 查询 | 用途 |
|------|------|
| `assignee = currentUser()` | 我的任务 |
| `project = PROJ AND status = "In Progress"` | 项目进行中的任务 |
| `priority in ("Highest", "High")` | 高优先级任务 |

## 培训方式

### 线上培训

**工具**：Zoom, Teams, Google Meet

**流程**：
1. 共享屏幕，演示操作
2. 参与者跟随操作
3. 实时答疑
4. 记录培训视频，供后续参考

### 线下培训

**流程**：
1. 现场演示
2. 手把手指导
3. 实操练习
4. 现场答疑

### 自学材料

**提供**：
- 入职指南文档
- 视频教程
- 常见问题 FAQ
- 最佳实践文档

## 培训检查清单

### 培训前

- [ ] 确认培训对象和人数
- [ ] 确认培训时长和内容
- [ ] 准备培训材料（PPT、文档）
- [ ] 预约培训时间和会议室
- [ ] 准备测试账号（如需要）

### 培训中

- [ ] 按计划执行培训内容
- [ ] 控制培训节奏，确保理解
- [ ] 互动答疑，及时解决疑问
- [ ] 记录培训反馈

### 培训后

- [ ] 发送培训材料
- [ ] 收集培训反馈
- [ ] 跟进培训效果
- [ ] 定期回顾和优化培训内容

## 最佳实践

1. **分阶段培训**：先基础后进阶，逐步深入
2. **实操为主**：演示 + 实操，边学边练
3. **答疑及时**：及时解答疑问，避免积累问题
4. **材料丰富**：提供多种形式的学习材料（文档、视频）
5. **定期回顾**：根据反馈定期优化培训内容
6. **建立答疑渠道**：培训后提供持续的答疑支持
